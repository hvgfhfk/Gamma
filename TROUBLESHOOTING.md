# 문제 해결 가이드

## 일반적인 문제

### 1. 감마/밝기/대비 조정이 적용되지 않음

#### 증상
- 슬라이더를 조정해도 화면이 변하지 않음
- 로그에 "SetDeviceGammaRamp 실패" 메시지

#### 해결 방법

**방법 1: 관리자 권한으로 실행**
1. `Gamma.exe` 우클릭
2. "관리자 권한으로 실행" 선택
3. 다시 시도

**방법 2: HDR 모드 확인**
- Windows 설정 → 시스템 → 디스플레이
- HDR 모드가 활성화되어 있으면 비활성화 시도
- 또는 NVIDIA 그래픽 카드 사용 시 HDR 모드에서도 정상 동작

**방법 3: 그래픽 드라이버 업데이트**
- NVIDIA: [GeForce Experience](https://www.nvidia.com/ko-kr/geforce/geforce-experience/)
- AMD: [AMD Software](https://www.amd.com/ko/support)
- Intel: [Intel Graphics Drivers](https://www.intel.com/content/www/us/en/download-center/home.html)

**방법 4: 다른 프로그램과의 충돌 확인**
- 다른 감마 조정 프로그램 종료
- 게임 오버레이 프로그램 확인 (예: Discord, Steam)

---

### 2. 프로그램이 시작되지 않음

#### 증상
- 더블클릭해도 아무 반응 없음
- 또는 즉시 종료됨

#### 해결 방법

**방법 1: .NET Framework 확인**
```
제어판 → 프로그램 → 프로그램 및 기능
".NET Framework 4.7.2" 또는 상위 버전 설치 확인
```

**방법 2: 로그 파일 확인**
- `error_log.txt` 파일 열기
- 최근 오류 메시지 확인

**방법 3: 다른 위치에서 실행**
- 현재 위치에서 실행 파일을 다른 폴더로 복사
- 새 위치에서 실행 시도

**방법 4: Windows 이벤트 뷰어 확인**
- `Win + R` → `eventvwr.msc`
- Windows 로그 → 응용 프로그램
- 최근 오류 이벤트 확인

---

### 3. 단축키가 작동하지 않음

#### 증상
- 단축키를 눌러도 반응 없음
- 다른 동작이 실행됨

#### 해결 방법

**방법 1: 단축키 충돌 확인**
- 다른 프로그램에서 같은 단축키 사용 중인지 확인
- 단축키 변경 시도

**방법 2: 단축키 재등록**
1. 설정 탭에서 단축키 해제
2. 프로그램 재시작
3. 단축키 다시 등록

**방법 3: 관리자 권한 확인**
- 일부 단축키는 관리자 권한 필요
- 관리자 권한으로 실행 후 시도

---

### 4. 프로세스 자동 감지가 작동하지 않음

#### 증상
- 프로세스가 활성화되어도 설정이 적용되지 않음

#### 해결 방법

**방법 1: 프로세스 이름 확인**
- 프로세스 추가 시 정확한 프로세스 이름 사용
- 작업 관리자에서 프로세스 이름 확인 (예: `notepad.exe`)

**방법 2: 프로세스 감지 활성화 확인**
- 프로세스 탭에서 "프로세스 감지 활성화" 토글 확인

**방법 3: 프로세스 규칙 재추가**
1. 기존 규칙 삭제
2. 프로세스 다시 추가
3. 설정 다시 입력

---

### 5. 자동 실행이 작동하지 않음

#### 증상
- 설정에서 자동 실행 체크했지만 시작 시 실행되지 않음

#### 해결 방법

**방법 1: 레지스트리 확인**
- `Win + R` → `regedit`
- `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- `Gamma_Controller` 항목 존재 확인
- 경로가 올바른지 확인

**방법 2: 자동 실행 재설정**
1. 설정에서 자동 실행 체크 해제
2. 프로그램 재시작
3. 다시 체크

---

### 6. 시스템 트레이 아이콘이 보이지 않음

#### 증상
- 프로그램을 최소화했지만 트레이 아이콘 없음

#### 해결 방법

**방법 1: 트레이 아이콘 표시 설정 확인**
- Windows 설정 → 개인 설정 → 작업 표시줄
- "작업 표시줄에 표시할 아이콘 선택" 확인

**방법 2: 프로그램 재시작**
- 프로그램 완전 종료 후 재시작

---

## 고급 문제 해결

### 로그 파일 분석

`error_log.txt` 파일을 열어 다음 키워드로 검색:

- **"SetDeviceGammaRamp 실패"**: 감마 조정 실패
  - 관리자 권한 필요 또는 드라이버 문제
  
- **"GetDC 실패"**: 디스플레이 컨텍스트 가져오기 실패
  - 그래픽 드라이버 문제 가능
  
- **"단조 증가 위반"**: 감마 램프 값 문제
  - 프로그램 내부 오류, 개발자에게 문의
  
- **"NvAPI"**: NVIDIA API 관련 오류
  - NVIDIA 드라이버 업데이트 필요

### Windows 이벤트 뷰어

1. `Win + R` → `eventvwr.msc`
2. Windows 로그 → 응용 프로그램
3. 오류 또는 경고 이벤트 확인
4. `Gamma.exe` 관련 이벤트 검색

### 안전 모드에서 테스트

1. Windows 안전 모드로 부팅
2. 프로그램 실행
3. 정상 동작 여부 확인
4. 안전 모드에서 동작하면 다른 프로그램과의 충돌 가능성

## 성능 문제

### 프로그램이 느리게 동작함

**해결 방법:**
1. 프로세스 규칙 개수 확인 (너무 많으면 성능 저하)
2. 불필요한 프로세스 규칙 삭제
3. 프로그램 재시작

### 메모리 사용량이 높음

**해결 방법:**
1. 작업 관리자에서 메모리 사용량 확인
2. 정상 범위: 50-100MB
3. 이상 시 프로그램 재시작

---

## 바이러스 백신 경고

### Windows Defender 또는 다른 백신에서 경고

**원인:**
- 신뢰할 수 없는 출처의 실행 파일
- 코드 서명 없음

**해결 방법:**

**Windows Defender:**
1. Windows 보안 → 바이러스 및 위협 방지
2. "바이러스 및 위협 방지 설정" → "설정 관리"
3. "제외" → "제외 추가 또는 제거"
4. `Gamma.exe` 파일 또는 폴더 추가

**다른 백신:**
- 각 백신 프로그램의 예외 목록에 추가
- 신뢰할 수 있는 출처에서 다운로드 확인

## 지원 요청

위 방법으로 해결되지 않으면 다음 정보와 함께 문의:

1. **오류 로그**: `error_log.txt` 파일
2. **시스템 정보**:
   - Windows 버전
   - 그래픽 카드 모델
   - .NET Framework 버전
3. **재현 단계**: 문제가 발생하는 정확한 단계
4. **스크린샷**: 가능한 경우
